## 五、getCHSMPk 返回的 algorithm 取值

> 位置：文档 7.3.1 getCHSMPk 详细定义中，result.algorithm 示例值为 "sm3"

**文档描述**：返回结果中 `algorithm` 字段描述为"指纹的HASH算法标识"，示例取值 `sm3`

**说明**：
1. 一期中指纹算法是 SHA256，二期示例只展示了 sm3
2. 既然二期同时支持 rsa 和 sm2 公钥，返回的 algorithm 应该也可能是 `sha256`

> ⚠️ **待确认**：当配置 rsa 公钥时，getCHSMPk 返回的 algorithm 是否为 `sha256`？用例中暂覆盖 sm3 和 sha256 两种断言

## 六、同 URL 多接口通过 oprType 区分

以下接口共享同一 URL，通过请求体中的 `oprType` 字段区分：

**URL: `/chsm`（POST）- 共6个操作：**
- getCHSMInfo (oprType=getinfo)
- upgradeCHSM (oprType=upgrade)
- restartCHSM (oprType=restart)
- backupCHSM (oprType=backup)
- restoreCHSM (oprType=restore)
- getCHSMDeviceInfo (oprType=getDeviceInfo)

**URL: `/vsm`（POST）- 共6个操作（不含预留）：**
- getVSMInfo (oprType=getinfo)
- startVSM (oprType=start)
- stopVSM (oprType=stop)
- restartVSM (oprType=restart)
- resetVSM (oprType=reset)
- upgradeVSM (oprType=upgrade)

**URL: `/chsm/image`（POST）- 共2个操作：**
- exportCHSM (oprType=export)
- importCHSM (oprType=import)

**URL: `/vsm/image`（POST）- 共2个操作：**
- exportVSM (oprType=export)
- importVSM (oprType=import)

> **确认项**：此设计已知晓，测试时需重点覆盖 oprType 为空、非法值、大小写敏感等异常场景。

## 八、回调机制（callbackUrl）的回调格式定义

以下接口使用了异步回调机制：
- exportCHSM / importCHSM（7.1.8 / 7.1.9）
- upgradeCHSM / restartCHSM（7.1.10 / 7.1.11）
- backupCHSM / restoreCHSM（7.1.12 / 7.1.13）
- exportVSM / importVSM（7.2.5 / 7.2.6）
- startVSM / stopVSM / restartVSM / resetVSM / upgradeVSM（7.2.7-7.2.13）

✅ **已确认**：
1. 不需要搭建回调服务，测试时提供一个 mock 地址即可
2. 仅验证主接口返回，不验证回调内容
3. callbackUrl 字段在用例中填写 mock 地址（如 http://10.10.1.207:8000/callback）

## 九、SM2WithSM3 签名算法适配范围

> 位置：文档 6.7 认证机制 中定义，CHSM-SignatureAlg 支持 "RSAWithSHA256 或 SM2WithSM3"

✅ **已确认**：
1. 所有 trusted 接口需要分别用两种签名算法运行（RSAWithSHA256 和 SM2WithSM3）
2. 在代码中通过配置切换签名算法，开启认证后配置哪种算法就使用对应的签名和请求头
3. 部分接口的 `alg` 参数（数据包签名）与认证头签名（CHSM-SignatureAlg）是独立的两层签名
