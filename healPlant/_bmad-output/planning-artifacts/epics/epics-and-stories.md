# Epics & Stories

## Epic 1: 项目基础设施搭建

**目标：** 初始化 Next.js 项目，配置技术栈核心依赖，建立数据库 Schema 和基础项目结构。

### Story 1.1: 项目初始化与核心依赖安装
**作为** 开发者，**我需要** 初始化 Next.js 项目并安装所有核心依赖，**以便** 后续功能开发有坚实的技术基础。

**验收标准：**
- Next.js 14 App Router 项目创建成功
- TypeScript 配置完成
- Tailwind CSS + shadcn/ui 安装配置
- Drizzle ORM + better-sqlite3 安装配置
- NextAuth.js v5 安装
- pnpm 作为包管理器
- ESLint + Prettier 配置
- 项目目录结构按架构文档建立
- `pnpm dev` 能成功启动开发服务器

### Story 1.2: 数据库 Schema 定义与迁移
**作为** 开发者，**我需要** 用 Drizzle ORM 定义全部数据库表结构并运行迁移，**以便** 应用有完整的数据持久层。

**验收标准：**
- 全部 16 张表的 Drizzle Schema 定义完成
- 外键关系正确建立
- 迁移文件生成并成功执行
- 数据库索引按需创建（varietyId, date 字段等）
- 可通过 drizzle-kit studio 查看数据库结构

### Story 1.3: 基础布局组件
**作为** 开发者，**我需要** 创建前台和后台的基础布局组件，**以便** 页面有统一的导航和外观框架。

**验收标准：**
- PublicLayout: 顶部导航栏（草绿底白字）+ 页脚
- AdminLayout: 侧边导航 + 顶部栏
- 移动端导航适配（前台汉堡菜单、后台抽屉菜单）
- 全局 CSS 变量定义（色彩体系、间距、圆角）
- 基础 shadcn/ui 组件引入（Button, Input, Dialog, Table, Card, Tabs）

## Epic 2: 认证与权限系统

**目标：** 实现管理员登录认证和前后台路由守卫，确保权限隔离。

### Story 2.1: 管理员登录功能
**作为** 管理员，**我可以** 通过账号密码登录系统，**以便** 访问后台管理功能。

**验收标准：**
- 登录页面 UI 完成（草绿主题）
- NextAuth.js Credentials Provider 配置
- bcrypt 密码验证
- JWT 存储在 httpOnly Secure Cookie
- 登录成功跳转 /admin
- 登录失败显示通用错误信息
- FR-1 通过

### Story 2.2: 路由守卫与 API 鉴权中间件
**作为** 系统，**我需要** 拦截未认证的后台访问请求，**以便** 保护私密数据不泄露。

**验收标准：**
- Next.js Middleware 拦截 /admin/* 路由
- 未登录访问后台路由重定向到 /login
- API Route 中间件校验 JWT
- 未携带有效 token 的 API 请求返回 401
- 管理员登出后 cookie 清除、无法访问后台
- FR-2, FR-3 通过

## Epic 3: 后台 — 品种资料库与智能定价

**目标：** 实现花草品种信息管理和双模式智能定价功能。

### Story 3.1: 品种资料库 CRUD
**作为** 管理员，**我可以** 新增、编辑、删除、查看花草品种档案，**以便** 系统化管理品种信息。

**验收标准：**
- 品种列表页：搜索、筛选（热度/难度/季节）、分页
- 新增品种表单：全字段录入
- 编辑品种：预填充已有数据
- 删除品种：二次确认
- 封面图上传
- FR-26 通过

### Story 3.2: 双模式智能定价
**作为** 管理员，**我可以** 对品种进行手动售价或成本涨幅比例定价，**以便** 灵活设定售价。

**验收标准：**
- 品种详情页中的定价面板
- 手动输入售价模式：直接输入并保存
- 涨幅比例模式：输入百分比，系统自动计算售价（= 进价 × (1 + 比例/100)）
- Radio 切换两种模式
- 定价历史列表展示
- FR-27, FR-28, FR-29, FR-30 通过

## Epic 4: 后台 — 进销存台账系统

**目标：** 实现进货、销售、损耗三大台账的 CRUD 和自动核算报表。

### Story 4.1: 进货台账
**作为** 管理员，**我可以** 录入和管理进货记录，**以便** 追踪采购成本。

**验收标准：**
- 进货记录列表：按时间范围筛选、分页
- 新增进货：选择品种（下拉关联品种库）、数量、单价、日期
- 总成本自动计算 = 数量 × 单价
- 支持编辑和删除
- FR-32 通过

### Story 4.2: 销售台账
**作为** 管理员，**我可以** 录入和管理销售记录，**以便** 追踪营收和利润。

**验收标准：**
- 销售记录列表：按时间范围筛选
- 新增销售：选择品种、数量、实际售价、日期
- 系统自动获取该品种最近进价作为成本基准
- 利润自动计算 = 总售价 - 成本基准 × 数量
- FR-33 通过

### Story 4.3: 损耗报废登记
**作为** 管理员，**我可以** 登记货品损耗报废，**以便** 追踪损耗成本。

**验收标准：**
- 损耗记录列表
- 新增损耗：品种、数量、原因选择（枯萎/破损/残次/其他）、日期
- 损耗成本自动计算 = 数量 × 最近进价
- FR-34 通过

### Story 4.4: 经营核算报表
**作为** 管理员，**我可以** 查看自动生成的经营报表，**以便** 快速了解盈亏状况。

**验收标准：**
- 时间范围选择（今日/本周/本月/自定义）
- 汇总指标卡片：总进货、总销售、总利润、毛利率、损耗量、损耗率
- 利润趋势折线图
- 品种销售占比饼图
- 品种利润排行表
- 所有数据零手动计算
- FR-35, FR-36 通过
