# PRD: HealPlant - 绿植综合经营网站

## 0. 文档目的

本 PRD 面向架构师、开发工程师和项目负责人，定义 HealPlant 绿植综合经营网站的全部功能需求。文档基于产品简报（`_bmad-output/planning-artifacts/briefs/brief-healPlant-2026-05-22/brief.md`）编写。功能按前台公开、后台私密两大区域组织，需求项使用全局 FR 编号。

## 1. 愿景

HealPlant 是一个为绿植经营个人从业者打造的综合网站，采用**前台公开展示 + 后台私密管理**的双面架构。前台是一个清新治愈的内容展示窗口——养花科普、趣味视频、服务介绍、产品展示——用于塑造专业个人品牌和线上获客引流。后台是一个私密的经营管理工具——品种选品、成本定价、进销存核算、盈亏分析——让个人经营者拥有精细化的数据决策能力。

两套界面共享同一数据底层，通过严格的角色权限隔离，确保经营隐私数据不会泄露给公开访客。整站采用草绿 + 暖白的园艺视觉风格，响应式适配手机和电脑浏览，操作门槛极低，面向零技术背景用户的日常使用。

## 2. 目标用户

### 2.1 主要角色

**管理员（站主本人）：** 武汉本地绿植兼职从业者，在鲜花小镇打工学习养护技术，同时经营地摊摆卖和上门养护服务，计划未来开实体花店。零代码技术背景，需要极低操作门槛。

**访客（匿名浏览者）：** 潜在养护服务客户、绿植购买意向顾客、养花爱好者、随机浏览者。

### 2.2 Jobs To Be Done

**管理员：**
- 我需要把每天学到的养护知识、进价信息、品种特征记录下来，方便以后选品拿货参考
- 我需要清楚每天摆摊赚了多少、亏了多少、哪些货损耗了，不想手动算
- 我需要灵活定价——有时直接定售价，有时按成本加成百分比来算
- 我需要一个对外展示窗口来接单、拓客，但不能让客户看到我的进价和成本
- 我需要模拟开店后的收支情况，评估是否值得开实体店

**访客：**
- 我需要找到靠谱的上门绿植养护服务并预约
- 我想看看有什么绿植可以买，了解怎么养
- 我想看有趣的养花日常内容打发时间

### 2.3 非用户（V1）

- 多名员工协作的花店团队（V1 仅支持单管理员）
- 需要在线支付/电商交易的买家
- 其他城市的服务客户（V1 限定武汉本地）

### 2.4 关键用户旅程

**UJ-1. 管理员录入今日进货并定价**
- **角色 + 场景：** 管理员从花卉市场进了一批货回来，需要录入进货信息并定好售价
- **入口状态：** 已登录后台，在进货台账页面
- **路径：** 点击"新增进货记录" → 填写品种名、数量、单价、进货时间 → 保存 → 跳转到该品种定价页面 → 选择"成本涨幅定价"模式 → 输入上浮 80% → 系统自动算出售价 → 确认保存
- **高潮：** 进货成本记入台账，售价自动计算完成，一步到位
- **结果：** 进货记录和定价记录均已存档，可在报表中查看

**UJ-2. 访客浏览并预约上门养护服务**
- **角色 + 场景：** 武汉某家庭用户搜索到网站，想找人上门给家里绿植做养护
- **入口状态：** 未登录，从搜索引擎进入前台首页
- **路径：** 浏览首页 → 点击养护服务页 → 查看服务项目和收费套餐 → 浏览案例对比图 → 填写在线预约表单（姓名、电话、地址、需求描述）→ 提交
- **高潮：** 预约表单提交成功，看到确认提示
- **结果：** 管理员后台收到预约信息

**UJ-3. 管理员查看本月经营报表**
- **角色 + 场景：** 月末管理员想了解本月摆摊经营的整体盈亏
- **入口状态：** 已登录后台，在报表页面
- **路径：** 选择时间范围"本月" → 查看总进货成本、总销售额、总利润、毛利率 → 查看损耗统计 → 查看可视化图表
- **高潮：** 一页看清本月经营全貌，无需手动计算
- **结果：** 管理员据此调整下月选品和定价策略

**UJ-4. 管理员发布一条视频动态**
- **角色 + 场景：** 管理员拍了一段绿植养护实操视频，想发到网站上
- **入口状态：** 已登录后台，在视频管理页面
- **路径：** 点击"上传视频" → 选择视频文件 → 填写标题、分类标签 → 设置为"公开可见" → 点击发布
- **高潮：** 视频上传成功，在前台趣味广场可直接播放
- **结果：** 访客在前台可看到并在线播放该视频

## 3. 术语表

- **前台** — 所有未登录访客可见的公开页面集合
- **后台** — 仅管理员登录后可访问的私密管理页面集合
- **管理员** — 唯一拥有后台权限的账号角色（V1 仅一个）
- **访客** — 未登录的匿名浏览者
- **品种** — 一种花卉或绿植类型，在品种资料库中有唯一记录
- **进货记录** — 一次采购行为的数据：品种、数量、单价、时间、总金额
- **销售记录** — 一次售卖行为的数据：品种、数量、售价、时间、利润
- **损耗记录** — 货品无法售卖的报废数据：品种、数量、原因、损耗成本
- **双模式定价** — 对同一品种可选手动输入售价或按成本涨幅比例自动计算售价的定价方式
- **涨幅比例** — 基于进货成本上浮的百分比，用于自动计算售价（售价 = 进价 × (1 + 涨幅比例/100)）
- **毛利率** — (总销售额 - 总进货成本) / 总销售额 × 100%
- **损耗率** — 损耗数量 / (售出数量 + 损耗数量) × 100%

## 4. 功能

### 4.1 认证与权限系统

**描述：** 网站的角色权限基础。管理员通过账号密码登录获得后台访问权限，未登录用户仅能浏览前台公开内容。所有后台路由和 API 端点必须进行鉴权校验，确保私密数据不泄露。实现 UJ-1, UJ-3, UJ-4。

**功能需求：**

#### FR-1: 管理员登录
管理员可通过账号密码登录系统，获得后台访问权限。

**可测试结果：**
- 正确凭据登录后跳转后台首页
- 错误凭据显示错误提示，不泄露是账号还是密码错误
- 登录状态通过 JWT/Session 维持，刷新页面不丢失

#### FR-2: 路由守卫
未登录用户访问后台路由时，自动重定向到登录页面。

**可测试结果：**
- 未登录用户直接访问 `/admin/*` 路径时重定向到登录页
- 后台 API 未携带有效 token 返回 401

#### FR-3: 管理员登出
管理员可主动登出，清除登录状态。

**可测试结果：**
- 登出后无法访问后台页面
- 登出后 token 失效

### 4.8 后台 — 花草品种信息资料库

**描述：** 后台核心数据库，录入所有绿植花卉的完整档案。包含公开属性和私密经营数据（进价、热度评级等），以及双模式智能定价功能。实现 UJ-1。

**功能需求：**

#### FR-26: 品种档案 CRUD
管理员可新增、编辑、删除、查看品种档案。每条档案包含：名称、外形特征、养护难度（易/中/难）、生长习性、适用场景、市场进价、市场欢迎热度等级（1-5星）、季节畅销指数（春/夏/秋/冬）、顾客喜爱反馈记录。

**可测试结果：**
- 所有字段可正常录入和展示
- 列表支持按名称搜索、按热度/难度/季节筛选
- 删除操作需二次确认

#### FR-27: 手动售价定价
管理员可直接输入某品种的拟定售卖价格。

**可测试结果：**
- 输入售价后保存，定价记录存档
- 显示该品种当前售价

#### FR-28: 成本涨幅比例定价
管理员可输入成本上浮百分比，系统根据该品种最新进价自动计算售价（售价 = 进价 × (1 + 涨幅比例/100)）。

**可测试结果：**
- 输入涨幅比例 80%，进价 10 元，计算结果为 18 元
- 计算结果显示并可保存

#### FR-29: 定价模式切换
管理员可在手动售价和涨幅比例定价之间自由切换。

**可测试结果：**
- 切换模式后输入区域相应变化
- 两种模式结果均可保存

#### FR-30: 定价历史记录
系统保存每次定价记录（时间、模式、定价参数、结果售价），可查看历史。

**可测试结果：**
- 定价修改后旧记录保留
- 可查看某品种的定价变更历史列表

- 可编辑和删除记录

#### FR-33: 销售台账
管理员可录入销售记录：售卖日期、品种、数量、实际售价。系统自动计算单笔利润（= 售价 - 该品种最近进价 × 数量）和每日营收。

**可测试结果：**
- 利润自动计算正确
- 日营收 = 当日所有销售记录金额合计

#### FR-34: 损耗报废登记
管理员可记录无法售卖的货品：品种、数量、原因（枯萎/破损/残次/其他）。系统自动计算损耗成本。

**可测试结果：**
- 损耗成本 = 数量 × 该品种最近进价
- 可按原因分类统计

#### FR-35: 自动核算报表
系统根据进货、销售、损耗数据，自动计算并展示：单日利润、月度利润、毛利率、损耗率、资金结余。

**可测试结果：**
- 选择日期范围后报表数据自动更新
- 毛利率 = (总销售 - 总进货成本) / 总销售 × 100%
- 损耗率 = 损耗数量 / (售出 + 损耗) × 100%

#### FR-36: 可视化数据图表
报表以图表形式展示关键指标趋势（折线图、柱状图、饼图）。

**可测试结果：**
- 利润趋势折线图正确反映日/月数据
- 品种销售占比饼图数据准确

### 4.11 后台 — 实体小店收支测算

**描述：** 模拟开店场景的收支核算工具，帮助管理员评估开店可行性。

**功能需求：**

#### FR-37: 成本项录入
管理员可录入固定成本（月租金、基础水电）和变动成本（耗材、人工、种苗囤货等），每项含名称、金额、频率（月/季/年）。

**可测试结果：**

- 支持不同频率自动折算为月均成本

#### FR-38: 营收预估录入
管理员可录入预估月营收数据。

**可测试结果：**

#### FR-39: 收支测算报表
系统自动核算：月总支出、营收差额、回本周期、年度盈利预估。

**可测试结果：**
- 月支出 = 所有成本项月均合计
- 营收差额 = 预估月营收 - 月支出

## 5. 非目标（明确排除）

- **不做在线支付/交易系统** — V1 为展示型 + 管理型网站，线下交易为主
- **不做多管理员/团队协作** — V1 仅支持单管理员账号
- **不做第三方平台对接** — 不集成微信公众号、抖音、小红书等
- **不做 SEO 优化和流量分析** — V1 聚焦核心功能
- **不做自动化营销** — 不含推送、优惠券、促销活动功能
- **不做在线客服/即时通讯** — 通过留言板和联系方式替代

## 6. MVP 范围

### 6.1 包含
- 管理员登录认证和角色权限隔离
- 前台全部 6 个公开板块
- 后台全部 6 个私密板块 + 预约管理
- 视频上传、在线播放、分类管理
- 双模式智能定价
- 进货/销售/损耗台账及自动核算报表
- 实体小店收支测算
- 库存管理
- 互动留言（含管理员回复）
- 响应式设计（移动端 + 桌面端）
- 清新治愈园艺风 UI（草绿 + 暖白）

### 6.2 MVP 不包含
- 在线支付功能 — 线下交易为主，V2 可考虑
- 多角色权限 — V1 单管理员，开店后再扩展
- 第三方集成 — V2 可对接微信生态
- 数据导出（Excel/PDF）— 用户未提出，V2 评估
- 多语言支持 — 面向本地用户，仅中文

## 7. 成功指标

**主要**
- **SM-1**: 管理员日常数据录入效率 — 单条进货 + 定价记录 ≤ 5 分钟完成。验证 FR-26, FR-27, FR-28, FR-32。
- **SM-2**: 经营数据零手动计算 — 利润、毛利率、损耗率、收支差额全部系统自动生成。验证 FR-35, FR-39。
- **SM-3**: 权限隔离完整性 — 未登录用户无任何途径访问后台数据。验证 FR-2。

**次要**
- **SM-4**: 视频功能完整性 — 视频可上传、播放、分类管理、上下架。验证 FR-41, FR-42, FR-43, FR-44。
- **SM-5**: 移动端体验 — 主要页面在 375px 宽度下布局正常、操作流畅。验证全部前台 FR。

**反指标（不应优化）**
- **SM-C1**: 内容发布频率 — 不应以发布数量为目标驱动质量下降。平衡 SM-4。
- **SM-C2**: 页面访问量 — V1 不以流量为核心目标，避免为流量牺牲功能完整性。

**性能：**
- 页面首次加载 ≤ 3 秒（国内网络环境）

- 操作门槛：零技术背景用户可在 10 分钟内学会基本操作
- 所有表单有明确的输入提示和错误提示
